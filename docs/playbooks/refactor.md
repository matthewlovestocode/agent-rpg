# Playbook: Refactor

## Routing
1. explorer
2. senior_dev
3. tester
4. reviewer
5. verifier

## Minimal handoff sequence
1. explorer: identify coupling and impact
2. senior_dev: perform scoped refactor
3. tester: run impacted test suite
4. reviewer: check maintainability/regression risk
5. verifier: confirm behavior parity

## Exit gates
- Behavior parity confirmed
- Tests pass or failures explained
- Tradeoffs and residual risks documented
