# Playbook: Feature Delivery

## Routing
1. explorer
2. worker or senior_dev
3. tester
4. verifier
5. documentation (if behavior/config/workflow changed)

## Minimal handoff sequence
1. explorer: map impact + unknowns
2. implementation agent: implement scoped changes
3. tester: run targeted tests and report gaps
4. verifier: validate acceptance criteria
5. documentation: align docs/examples

## Exit gates
- Acceptance criteria checked
- Test status known
- Risks documented
